## 为什么只选 20 个修改案例

首版不追求把每条检查规则都变成任务，而是覆盖用户最常遇到、也最容易被 AI 误伤的修改：

- 改一个日期但不动全稿。
- 修复越界、重叠和密集信息，同时保留业务含义。
- 处理截图说明和连续操作流程，避免图片变形。
- 修复可编辑表格，避免截图化。
- 清理备注、个人联系方式和外部文件依赖。
- 在多页复杂原稿中保持非目标页完全不变。

后续新增案例必须回答三个问题：客户真实会不会这样改、现有工具是否经常改乱、结果能否通过机器证据和人工视觉复核共同验收。仅为了增加题量的案例不进入主集。
